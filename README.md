# The Recursive I Ching

The third site in the recursive.eco family, after [The Recursive Tarot](https://tarot.recursive.eco)
and [The Recursive Astrology](https://astro.recursive.eco) — the **same creed and method, a
different domain**. A static, backend-free site that turns a folder of JSON into a browsable
library of the I Ching: the 64 hexagrams in full, a contemplative reading practice built from
them, and a cross-lens synthesis against Human Design gates, zodiac signs, and chakras.

**Read a hexagram to know yourself, not to be told your fate.** A hexagram is a mirror, not a
command — the same creed the sibling sites teach, pointed at a new subject.

## What's here

```
recursive-starter/
├── index.html                     the gallery homepage — every grammar, every view
├── theme.css                      the single source of colour (light only, cinnabar accent)
├── site-header.js / site-footer.js / view-switcher.js / icons.js / assistant.js
├── grammars/
│   ├── i-ching-summarized/        the flagship: all 64 hexagrams, Judgment/Image/lines/interpretation
│   ├── repair-iching/             12 hexagrams as a contemplative repair practice
│   ├── three-lenses-64/           hexagrams cross-read via Human Design gates / signs / chakras
│   └── _collection.json           generated by scripts/build_collection.py — never hand-edit
├── viewers/                       cards, explorer, lenses, tree-viewer, timeline — one grammar
│                                  or the whole collection, several ways to look
├── course/three-lenses.manifest.json + pages/course-viewer.html
│                                  the Three Lenses reading, chapters read live from three-lenses-64
├── scripts/build_collection.py    rebuild grammars/_collection.json after adding/editing a grammar
├── recursive-eco.json             channel manifest (how recursive.eco imports this repo)
├── ids.json                       slug -> recursive.eco grammar UUID (skeleton — nothing published yet)
└── GRAMMAR_FORMAT.md              grammar-JSON format reference — mirrored copy; canonical version is
                                    recursive.eco-schemas/GRAMMAR_FORMAT.md
```

## This repo is a playbook experiment

Built by following [`recursive-tarot/docs/REPLICATE-THE-PATTERN.md`](https://github.com/PlayfulProcess/recursive-tarot/blob/main/docs/REPLICATE-THE-PATTERN.md)
as the sole process authority — copy the working file, change data/branding/color/content, never
structure. See [`docs/PLAYBOOK-FIELD-REPORT.md`](docs/PLAYBOOK-FIELD-REPORT.md) for a specific,
honest account of where that doc was sufficient, where it was ambiguous or wrong, and the exact
edits it needs for the next site.

## Status

Content lifted from `recursive.eco-schemas`, chrome ported from `recursive-astrology`/
`recursive-tarot`, verified locally with Playwright. **Not yet published to recursive.eco**
(`ids.json` is an intentional empty skeleton) and not yet live on a custom domain (`CNAME` points
at `iching.recursive.eco`, DNS/Pages settings are a separate handoff step). See `CHANGELOG.md` for
the full build record.
